# airpods-volume-restore

에어팟(블루투스 오디오)이 연결되거나 아이폰→맥으로 핸드오프될 때 macOS가 출력 볼륨을
16단계 중 절반(약 50%)으로 리셋해버리는 오래된 버그가 있다. 이 스크립트는 에어팟으로
듣는 동안 볼륨을 기억해뒀다가, 리셋 직후 마지막 볼륨으로 되돌린다.

## 동작
- 에어팟이 기본 출력일 때 볼륨 변화를 3초 안정 후에만 `hs.settings`에 커밋(디바운스)
- 재연결/핸드오프로 에어팟이 기본 출력이 되면 1.5초 뒤 저장값으로 복원
- 복원 중 볼륨 이벤트는 `restoring` 플래그로 무시

## 디바운스가 필요한 이유
핸드오프 시 macOS의 50% 리셋 이벤트가 장치 변경(AUDIO) 이벤트보다 **먼저** 도착해서
좋은 값을 즉시 덮어쓰는 레이스가 있다. 볼륨 커밋을 3초 지연시키고, 리셋이 감지되면
대기 중인 커밋을 취소해서 리셋값이 저장을 오염시키지 못하게 한다.

## 튜닝
복원이 안 먹으면 `hs.timer.doAfter(1.5, ...)`의 `1.5`를 키운다. (macOS 리셋 타이밍 보정 knob)

## 설치
`airpods-volume-restore.lua` 내용을 `~/.hammerspoon/init.lua`에 붙여넣고 Reload Config.
